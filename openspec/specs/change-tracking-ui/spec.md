## Purpose
Интерфейс отслеживания изменений отображает список изменений с прогрессом, детальную карточку изменения с кнопками действий и группами задач, а также диалог подтверждения удаления.

## Requirements

### Requirement: Отображение строки изменения в списке
Система SHALL отображать каждую строку изменения с именем, индикатором статуса (зелёный для завершённых, жёлтый для активных), количеством задач и прогресс-баром.

#### Scenario: Активное изменение в списке
- **WHEN** изменение имеет незавершённые задачи
- **THEN** строка отображает зелёную точку перед именем, подсказку с общим числом задач (если есть группы) и прогресс-бар с текущим прогрессом

#### Scenario: Завершённое изменение в списке
- **WHEN** все задачи изменения завершены
- **THEN** строка отображает жёлтую точку перед именем и прогресс-бар, заполненный полностью

### Requirement: Подсветка и выбор строки при наведении
Система SHALL подсвечивать строку изменения при наведении курсора и обрабатывать клик для перехода к детальному виду.

#### Scenario: Наведение курсора на строку
- **WHEN** пользователь наводит курсор на строку изменения
- **THEN** фон строки меняется на приглушённый цвет, а текст прогресс-бара переключается на основной цвет

#### Scenario: Клик по строке
- **WHEN** пользователь кликает по строке изменения
- **THEN** вызывается обработчик выбора с именем изменения для открытия детального вида

### Requirement: Заголовок детальной карточки и кнопка возврата
Система SHALL отображать заголовок «Active Change» или «Completed Change» в зависимости от статуса и кнопку возврата к списку.

#### Scenario: Открытие активного изменения
- **WHEN** детальная карточка открывается для незавершённого изменения
- **THEN** заголовок показывает «Active Change» жёлтым цветом, а справа отображается кликабельная кнопка «← back»

#### Scenario: Открытие завершённого изменения
- **WHEN** детальная карточка открывается для завершённого изменения
- **THEN** заголовок показывает «Completed Change» зелёным цветом

### Requirement: Кнопки действий Apply, Update, Delete
Система SHALL предоставлять три кнопки действий под прогресс-баром: Apply, Update и Delete.

#### Scenario: Нажатие кнопки Apply
- **WHEN** пользователь нажимает кнопку «Apply»
- **THEN** отправляется команда `/opsx-apply <имя изменения>`

#### Scenario: Нажатие кнопки Update
- **WHEN** пользователь нажимает кнопку «Update»
- **THEN** отправляется команда `/opsx-update <имя изменения>`

#### Scenario: Нажатие кнопки Delete
- **WHEN** пользователь нажимает кнопку «Delete»
- **THEN** вместо кнопок действий появляется диалог подтверждения удаления

### Requirement: Диалог подтверждения удаления
Система SHALL показывать встроенный диалог подтверждения с предупреждением и кнопками Delete/Cancel перед удалением изменения.

#### Scenario: Подтверждение удаления
- **WHEN** пользователь нажимает «Delete» в диалоге подтверждения
- **THEN** вызывается обработчик удаления по имени изменения, после чего происходит возврат к списку

#### Scenario: Отмена удаления
- **WHEN** пользователь нажимает «Cancel» в диалоге подтверждения
- **THEN** диалог закрывается и снова отображаются кнопки действий Apply/Update/Delete

### Requirement: Рендеринг групп задач
Система SHALL рендерить группы задач детальной карточки с опциональным заголовком группы и списком задач, отмечая завершённые задачи галочкой.

#### Scenario: Завершённая задача в группе
- **WHEN** задача имеет статус done=true
- **THEN** перед текстом задачи отображается зелёная галочка «✓», а текст задачи показан приглушённым цветом

#### Scenario: Незавершённая задача в группе
- **WHEN** задача имеет статус done=false
- **THEN** перед текстом задачи отображаются два пробела, а текст задачи показан основным цветом

#### Scenario: Заголовок группы
- **WHEN** группа имеет заголовок (group.title)
- **THEN** заголовок отображается над задачами; если все задачи в группе завершены — заголовок показан приглушённым цветом, иначе — акцентным
